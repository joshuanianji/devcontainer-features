# Contributing

## Layout

Each feature lives in two directories:

- `src/<id>/` — the feature itself: `devcontainer-feature.json`, `install.sh`, optional `oncreate.sh`, and `NOTES.md`. `README.md` here is generated — don't edit it by hand.
- `test/<id>/` — `test.sh` (the autogenerated test) and `scenarios.json` plus a `<scenario>.sh` script per scenario.

## Add a feature

1. Create `src/<id>/` with a `devcontainer-feature.json` (`id`, `version`, `name`), an `install.sh`, and a `NOTES.md`.
2. Create `test/<id>/` with `test.sh` and `scenarios.json`.
3. Open a PR. `validate.yml` and `test.yaml` run automatically against the changed feature.
4. Merge to `main`.
5. Run the **Release & Docs** workflow manually (Actions → Release & Docs → Run workflow). It publishes the feature to `ghcr.io/joshuanianji/devcontainer-features/<id>` and opens an "Automated documentation update" PR.
6. Merge that docs PR.
7. Add a row for the feature to the table in the root `README.md` (this part is manual).

## Update a feature

1. Make the change in `install.sh` / the feature.
2. Bump `version` in `devcontainer-feature.json` (semver) — publishing only pushes new versions.
3. Add a row to the Changelog table in `NOTES.md`.
4. PR → merge → run **Release & Docs** → merge the docs PR.

## Docs

Each feature's `README.md` is generated from `devcontainer-feature.json` plus `NOTES.md`. Put all prose — usage, notes, changelog — in `NOTES.md`; anything written directly in `README.md` is overwritten on the next release.

## Deprecating

Set `"deprecated": true` in the feature's `devcontainer-feature.json`. CI (`test.yaml`) skips deprecated features.

## Testing locally

```sh
devcontainer features test -f <id> .          # autogenerated + scenarios
devcontainer features test -f <id> --skip-autogenerated .   # scenarios only
```
